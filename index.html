Introduction

Objectifs du Cours

  • Le cours explore les deux langages de balisage les plus courants (HTML / CSS) ainsi que leur potentiel éditorial, expérimenté d'abord comme un espace autonome (création de sites ex-nihilo) puis au sein d'une structure WordPress (site Spo).
  • Approche créative et expérientielle - le processus (le parcours) est aussi important que la destination.

Évaluation

Deux parties :

  • Un site web
  • Un carnet expliquant votre processus (peut être intégré au site web si vous le souhaitez)

Le site web

  • Créez un site web (publiquement visible, ou au moins partageable).
  • Le contenu peut porter sur n'importe quel sujet - blog, annonce d'événement, actualité, analyse, portfolio, etc.
  • Incluez différents éléments - texte, liens, titres, images, et éventuellement une vidéo ou un formulaire intégré.
  • Plusieurs pages, avec des liens entre elles et une page d'accueil.

Le carnet

  • Vos objectifs pour le projet
  • Les défis que vous avez rencontrés et comment vous les avez surmontés
  • Les choix que vous avez faits et pourquoi
  • Une auto-évaluation de votre projet mettant en lumière ce qui est bon mais aussi ce qui pourrait être amélioré

Méthodes / Principes

  • Expérimentation
    • Créativité, volonté d'essayer et d'échouer, recherche d'informations auprès d'experts...
    • Échouer rapidement. Boucle d'itérations rapides.
  • Comprendre / Repenser à partir des principes fondamentaux
  • Dépannage
    • Isoler les éléments, éliminer les sources possibles étape par étape...
  • Construire / Apprendre en public

Objectifs d'Apprentissage Appliqués

  • Par exemple, disons que vous voulez créer un site web. Comment pourriez-vous le faire ?
  • Disons que vous voulez apprendre Canva, Gimp, etc. Comment pourriez-vous procéder ?
  • Traduction par IA, génération de contenu par IA : Limites des LLM - hallucinations, etc.
  • Dites-moi ce que vous aimeriez apprendre !

Plateformes pour créer des sites web

  • Recommandé : Replit (attention - non publiquement visible sans carte de crédit, mais peu coûteux)
  • WordPress : l'approche 'standard'. Mieux si vous souhaitez inclure des commentaires, mais inclut des marques et de nombreux add-ons payants.
  • Autres plateformes : Wix, Weebly, Shopify pour le commerce, etc.
  • Plus avancé : GitHub Pages ; Vercel - nécessite de travailler avec du code.
